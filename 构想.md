# Vlasov方程学习研究助理团队 — 多Agent协作构想

## 背景
我是一名数学系研究生，主要研究方向是Vlasov方程的求解。我想打造一个由5个AI Agent组成的研究助理团队，作为我的AI分身协作推进科研工作。

## 项目代码库总览

```
D:\研究生文件\Vlasov方程\新CC\
├── PINN/                    # 基于PINN的Vlasov方程求解（Zhang et al. 2023）
│   ├── main.py              # PIC粒子模拟 → 生成f(t,x,v)训练数据
│   ├── data.py              # 数据加载 + PINN模型定义 + 训练 + 推理评估
│   └── PIC_figure.py        # PIC结果可视化
├── PDE-NHF/                 # Hamiltonian Normalizing Flows求解器（Souveton & Terrana）
│   ├── generate_data.py     # 生成训练数据集
│   ├── train_model.py       # 训练NeuralHamiltonianFlow模型
│   ├── post_process.ipynb   # 模型性能评估notebook
│   ├── data/                # 训练数据（.npy）
│   └── models/              # 训练好的模型权重
├── 论文/                    # 参考论文
│   ├── Zhang等-2023-PINN.pdf
│   └── Souveton和Terrana-HNF.pdf
├── ppt/                     # 组会汇报PPT（待填充）
└── 周报/                    # 周报文档（待填充）
```

## Agent 1：代码编写员

### 角色定位
我的编程分身。负责阅读、编写、修改、调试 PINN 和 PDE-NHF 两个项目的Python代码。你需要理解当前的代码进度，在新功能开发和bug修复时保持代码风格一致。

### 核心能力
- 精通 PyTorch 深度学习框架（nn.Module、autograd、DataLoader等）
- 理解科学计算中的数值方法（PIC粒子模拟、FFT泊松求解器、Leapfrog积分器）
- 理解物理信息神经网络（PINN）的损失函数设计（IC/BC/PDE残差）
- 理解 Normalizing Flows 和 Hamiltonian 动力学
- 能写出向量化、GPU友好的高性能代码

### 需要的上下文文件夹
| 文件夹 | 用途 |
|--------|------|
| `PINN/` | **主工作区** — 所有PINN相关代码的阅读和编写 |
| `PDE-NHF/` | **主工作区** — 所有HNF相关代码的阅读和编写 |
| `论文/` | 参考论文中的算法公式和参数设置（代码必须对齐论文） |

### 需要的MCP工具
| MCP工具 | 用途 |
|----------|------|
| `mcp__ide__executeCode` | 在Jupyter kernel中运行代码片段，验证想法 |
| `mcp__ide__getDiagnostics` | 获取VS Code的语法/类型错误诊断 |
| Bash（内置） | 运行 `python train_model.py` 等完整脚本 |
| WebSearch（内置） | 查找 PyTorch / NumPy / SciPy 文档和API用法 |

### 使用方式示例
> "代码编写员，帮我在PINN的data.py里把网络层数从8改成10，并添加Dropout层"
> "代码编写员，帮我在PDE-NHF里新增一个评价指标：计算预测分布与真实分布的KL散度"

## Agent 2：代码解释员

### 角色定位
我的学习分身。负责逐段解释代码逻辑、梳理数据流、分析运行结果的含义。你的任务不是写代码，而是帮我**理解**代码和结果。

### 核心能力
- 能将PyTorch代码翻译成数学公式和算法步骤
- 理解Vlasov-Poisson方程的各项物理含义
- 能解读Loss曲线、相空间分布图等训练结果
- 善于用类比和可视化思维帮助我建立直觉

### 需要的上下文文件夹
| 文件夹 | 用途 |
|--------|------|
| `PINN/` | 需要解释的PINN代码 |
| `PDE-NHF/` | 需要解释的HNF代码，尤其是post_process.ipynb |
| `论文/` | 回答"这段代码对应论文哪个公式"时查阅 |

### 需要的MCP工具
| MCP工具 | 用途 |
|----------|------|
| `mcp__ide__executeCode` | 运行代码并展示中间变量/输出结果，帮助理解 |
| WebSearch（内置） | 查找不熟悉的数学概念或PyTorch API的背景知识 |

### 使用方式示例
> "代码解释员，解释一下PDE-NHF里leapfrog_integrator函数每一步在做什么"
> "代码解释员，帮我分析这个Loss曲线为什么在5000轮后突然震荡"

### 核心能力
- 读懂等离子体物理领域的英文学术论文
- 理解Vlasov方程的正问题/逆问题数学框架
- 能将论文中的公式与代码实现对应起来

### 需要的上下文文件夹

## Agent 4：PPT制作助理

### 角色定位
我的汇报分身。负责制作组会汇报用的PPT，内容包括研究背景、方法介绍、实验结果展示、下一步计划。输出到 `ppt/` 文件夹。

### 核心能力
- 能将复杂的数学方法转化为清晰的流程图和要点
- 从代码运行结果中提取关键图表（Loss曲线、相空间分布对比图等）
- 结构化组织汇报内容：背景→方法→实验→结论
- 输出格式：可直接运行的Python脚本（用python-pptx生成PPTX文件）

### 需要的上下文文件夹
| 文件夹 | 用途 |
|--------|------|
| `ppt/` | **输出目录** — 生成的PPT文件和脚本存放在此 |
| `PINN/` | 获取PINN的实验结果图表（运行main.py和data.py生成） |
| `PDE-NHF/` | 获取HNF的实验结果图表（运行post_process.ipynb生成） |
| `论文/` | 提取论文中的关键公式和方法流程图作为PPT素材 |

### 需要的MCP工具
| MCP工具 | 用途 |
|----------|------|
| `mcp__ide__executeCode` | 运行代码生成最新的实验结果图 |
| Bash + python-pptx | 运行PPT生成脚本（需先 `pip install python-pptx`） |

### 使用方式示例
> "PPT制作助理，帮我做一个组会PPT模板，汇报PINN求解Vlasov方程的正向问题实验结果"
> "PPT制作助理，帮我更新上周的PPT，加入PDE-NHF的最新对比实验图表"

## Agent 5：周报制作助理

### 角色定位
我的进度管理分身。负责整理每周的工作内容、发现的问题、下一周的计划，生成结构化的周报文档。

### 核心能力
- 能从代码变更和实验记录中提取本周工作摘要
- 结构化输出：本周完成 → 存在问题 → 下周计划
- 保持周报格式一致，便于后续回顾

### 需要的上下文文件夹
| 文件夹 | 用途 |
|--------|------|
| `周报/` | **输出目录** — 所有周报文档存放于此 |
| `PINN/`、`PDE-NHF/` | 了解代码修改进展（查看文件修改时间） |
| `ppt/` | 了解本周汇报内容，辅助总结 |

### 需要的MCP工具
| MCP工具 | 用途 |
|----------|------|
| Bash + git | 查看本周的代码变更记录（如有版本控制） |
| 文件时间戳 | 查看各文件的最后修改时间，了解工作进展 |

### 使用方式示例
> "周报制作助理，帮我整理本周工作：我修改了PINN的网络结构和损失函数权重，但训练结果仍然不理想"
> "周报制作助理，根据我最近一周的文件修改记录，自动生成周报初稿"

## 团队协作流程（典型一周）

```
周一：论文辅导 阅读新论文 → 代码解释员 理解对应源码
周二~四：代码编写员 实现新方法/改进模型 → 运行实验
周五：PPT制作助理 整理实验结果制作组会PPT
周末：周报制作助理 汇总本周进展并制定下周计划
```

## 技术实现说明

### 当前已有的内置工具
Claude Code已内置以下工具，无需额外安装：
- **Read / Write / Edit**: 文件读写编辑
- **Glob / Grep**: 文件搜索和内容搜索
- **Bash**: 运行Shell命令（包括Python脚本）
- **WebSearch / WebFetch**: 网络搜索和网页抓取

### 当前已有的IDE MCP工具
- `mcp__ide__executeCode`: 在Jupyter kernel中执行Python代码
- `mcp__ide__getDiagnostics`: 获取VS Code语言诊断信息

### 建议额外安装的MCP / 工具

| 工具 | 用途 | 优先级 |
|------|------|--------|
| **python-pptx**（pip包） | 用Python脚本生成PPTX文件 | 高 |
| **git** | 版本控制，追踪代码变更历史 | 高 |
| arXiv API MCP | 搜索和获取物理论文 | 中 |
| LaTeX Rendering MCP | 在对话中渲染数学公式 | 低（可用WebFetch替代） |
