# PDE-NHF 方法求解双流不稳定性 — 可行性分析与实施计划

## 一、可行性结论

**原则上可行，但需充分认识到 PDE-NHF 和 PINN 两种方法的根本范式差异。**

PINN 是"单一初值问题求解器"——对一种初始条件训练一个网络。PDE-NHF 是"替代模型"——从大量不同初始条件的 PIC 模拟中学习通用的哈密顿势能 V(q)，然后用它替代 PIC 求解器。因此 PDE-NHF 不能直接"搬过来用"，需要改造数据生成和先验分布。

核心风险：双流不稳定性产生的相空间 filamentation（细丝化结构）对光滑神经网络构成挑战，且长时间演化需要大量蛙跳步数，梯度反传可能不稳定。

---

## 二、关键差异对比

| 特性 | PINN (当前) | PDE-NHF (当前) | 双流 + HNF (目标) |
|------|------------|---------------|-------------------|
| 初始条件 | 双流: p ~ 0.5N(+1,0.02²)+0.5N(-1,0.02²) | 单高斯: p ~ N(0, σ_p²) | 需改为双流 |
| 空间分布 | 均匀 U(0, L) | 高斯 N(64, σ_q²) | 需改为均匀或弱扰动 |
| 模拟时长 | t=62.5 (500步, dt=0.125) | t=1.0 (25步, dt=0.04) | 需大幅增加 |
| 粒子数 | 100,000 | 256 | 需增加 |
| 训练样本数 | 1 个初值 | 32,768 个不同初值 | 需生成大量变参数样本 |
| 先验分布 | N/A (PINN不用) | p0~N(0,σ_p²), q0~N(64,σ_q²) | p0 需改为混合高斯 |
| 条件向量维度 | N/A | 2 (σ_q, σ_p) | 需扩展到 3-5 维 |

## 三、实施计划

### 阶段 A：数据生成改造 (`generate_data.py`)

**A1. 初始条件改为双流分布**

将粒子初始化从单高斯改为双流:
- 一半粒子: p ~ N(+v_stream, v_spread²), 另一半: p ~ N(-v_stream, v_spread²)
- 空间分布: q ~ U(0, L) + 小幅正弦扰动(可选, 用于种子不稳定性)
- 固定的离子中和背景 (密度=1)

**A2. 参数空间设计（让 HNF 有足够多样性学习）**

变化参数作为 cond 向量:
- `v_stream` ∈ [0.8, 1.5] — 束流速度 (决定不稳定性增长率)
- `v_spread` ∈ [0.01, 0.05] — 热速度展宽 (决定稳定性阈值)
- `A_perturb` ∈ [0.001, 0.01] — 空间扰动幅度 (种子噪声)
- 总计: cond 维度 3

**A3. PIC 模拟参数调整**

- 域大小 L: 保持与 PINN 一致的缩放。PINN 用 L=10λ_D。但 PDE-NHF 用 L=128。需统一。
  - **建议**: 使用 L=10 (与 PINN 一致, 便于对比验证), Nx=128, N_PARTICLES=1000
- 时间: 先做短时间 tEnd=10 (覆盖线性增长阶段), 后续再扩展
- dt=0.1, 总步数=100, 保存间隔=10步
- 蛙跳步数 L_steps=100 (与 PIC 一致, 或适当减少)

**A4. 数据量**

- 先做小规模验证: N_EXAMPLES=1000, 变参数采样
- 成功后扩展到 N_EXAMPLES=10000+

### 阶段 B：模型改造 (`train_model.py`)

**B1. 先验分布改为混合高斯 (p) + 均匀 (q)**

```python
# 原: p0 ~ N(0, sigma_p)
# 新: p0 ~ 0.5 * N(+v_stream, v_spread²) + 0.5 * N(-v_stream, v_spread²)
# 使用 D.MixtureSameFamily 或手动计算 log_prob

# 原: q0 ~ N(64, sigma_q)  
# 新: q0 ~ U(0, L)  (在周期性域上的均匀分布)
# log_prob = -log(L) 常数 (可忽略, 因为与参数无关)
```

**B2. 条件向量扩展**

原: `sigma_q, sigma_p = cond[:,0], cond[:,1]`
新: `v_stream, v_spread, A_perturb = cond[:,0], cond[:,1], cond[:,2]`

**B3. 势能网络架构**

现有的 Deep Set 架构（每粒子嵌入 → 求和池化 → 全局 MLP）保持不变。它已经是置换不变和近似平移不变的，适用于双流场景。

可选增强: 将 cond 向量注入势能网络（类似 FiLM  conditioning），让网络学习参数化的势能。

**B4. 训练超参数调整**

- 蛙跳步数 L: 100 (匹配 PIC 步数)
- dt: -0.1 (反向积分, 与 PIC dt 一致)
- Batch size: 128 (不变)
- Epochs: 500 (更长演化需要更多训练)
- Learning rate: 0.0003 (不变)

### 阶段 C：评估改造 (`post_process.ipynb`)

- 相空间图: p 范围扩大到 [-3, 3] 以覆盖两个束流
- 新增诊断: 电场能量时间演化、不稳定性增长率对比
- 与 PIC ground truth 的定量对比 (Wasserstein-1 距离)
- 与 PINN 结果的定性对比

## 四、风险与缓解

| 风险 | 严重度 | 缓解措施 |
|------|--------|---------|
| 蛙跳步数过多导致梯度消失/爆炸 | 高 | 先用较少步数(50)验证; 考虑梯度裁剪; 使用检查点技术 |
| filamentation 超出网络表达能力 | 中高 | 先做短时间(线性阶段); 增加粒子数; 评估时关注小尺度结构 |
| 均匀空间先验与高斯先验差异大, 训练不稳定 | 中 | q 方向先验可保留弱高斯约束(N(μ_q, large_σ))作为正则化 |
| 单初始条件不足以训练 NF | 中 | 通过参数变化(阶段A2)产生多样性; 这是 NF 方法的内在需求 |
| 计算资源需求大(大量 PIC 模拟) | 中 | 减少粒子数(1000); 先做小规模验证; 可并行生成数据 |

---

## 五、验证方案

1. **单元验证**: 生成 10 个样本, 确认双流 PIC 代码正确 (相空间图呈现预期的不稳定性)
2. **训练验证**: 小规模训练 (1000 样本, 50 蛙跳步, 50 epochs), 确认 loss 下降
3. **物理验证**: 对比 HNF 预测 vs PIC ground truth 的相空间分布, 计算 W1 距离
4. **对比验证**: 将 HNF 结果与 PINN 方法的结果进行定性对比 (在 t=10,20,30 等时间点)
5. **诊断验证**: 检查哈密顿能量守恒 (HNF 的天然优势)
